# Дизайн сайта «Сообщества совместной практики» (xenitch.ru)

**Дата:** 2026-07-14
**Статус:** реализовано, сайт живой — https://xenitch.ru (2026-07-15; HTTPS enforced, почтовые записи сохранены)

## Цель

Сайт на домене xenitch.ru с двумя ролями:

1. **Корень** — витрина живых сообществ совместной практики, которые запускает Ксения (первое — писательская группа). Пока — текстовая заглушка-анонс.
2. **Раздел `/research/`** — презентация исследования «Ощутимая практика и неизбежность повторения»: рамка семи механизмов + статьи-разборы кейсов. Раздел будет пополняться новыми статьями.

## Структура и адреса

| Адрес | Страница |
|---|---|
| `xenitch.ru/` | Главная «Сообщества совместной практики»: короткий анонс (готовится писательская группа), ссылка на исследование |
| `xenitch.ru/research/` | Рамка исследования: манифест (почему одни практики повторяются, другие забрасываются), семь механизмов, карточки статей |
| `xenitch.ru/research/inklings/` | Статья про Инклингов (из `results/article-inklings-mechanisms.ru.md`) |
| `xenitch.ru/research/toastmasters/` | Статья про Toastmasters (из `results/article-toastmasters-mechanisms.ru.md`) |

Новые статьи добавляются как `xenitch.ru/research/{slug}/`.

Отдельной страницы «О проекте» в v1 нет: подпись автора живёт в подвале каждой страницы и в шапке раздела исследования.

- Язык сайта: русский (`lang="ru"`), включены CSS-переносы.
- Подпись: **Ксения Костюченко**, телеграм **@xenitch** (ссылка `https://t.me/xenitch`).

## Технический подход

Собственный минимальный генератор статики на Python (вариант выбран из трёх: свой генератор / Eleventy-Hugo / конструктор).

- Контент — markdown в `content/`, шаблоны — Jinja2 в `templates/`, сборка — `build.py`.
- Python работает в project-local `.venv` (конвенция проекта). Зависимости: `markdown` (или `markdown-it-py`), `jinja2`.
- Результат сборки кладётся в папку `docs/` репозитория — GitHub Pages умеет раздавать сайт именно из `/docs` ветки `main`. Дизайн-доки поэтому живут в `_specs/`, чтобы не смешиваться с выходом сборки.
- Никакого JavaScript на страницах. Только HTML + один CSS-файл + самостоятельно размещённые шрифты (woff2).

### Раскладка репозитория `github.com/xenitch/practice-site`

```
site/
├── _specs/                  # дизайн-доки (этот файл)
├── content/
│   ├── index.md             # главная (заглушка-анонс)
│   ├── research/
│   │   ├── index.md         # рамка: манифест + семь механизмов
│   │   ├── inklings.md
│   │   ├── toastmasters.md
│   │   └── impressionists.md
├── templates/
│   ├── base.html            # общий каркас: шапка, подвал
│   ├── page.html            # главная и research/index
│   └── article.html         # страница статьи
├── static/
│   ├── style.css
│   └── fonts/*.woff2
├── build.py                 # сборка content+templates → docs/
├── docs/                    # собранный сайт (коммитится, его раздаёт GitHub Pages)
│   └── CNAME                # xenitch.ru
└── README.md                # как собрать и опубликовать
```

Frontmatter статьи: `title`, `subtitle`, `date`, `description` (для карточки на /research/ и meta description). Список статей на `/research/` генерируется из frontmatter автоматически — отдельный индекс руками не ведём.

## Типографика и стиль («английский клуб», old money)

- **Бумага:** слоновая кость (#f5f0e6 ± подбор на превью); **чернила:** тёмный тепло-коричневый (почти чёрный).
- **Акценты:** гоночный зелёный (racing green) для ссылок и рубрик, бордо — второстепенный; тонкие золотисто-охристые линейки.
- **Шрифты** (полная кириллица, self-hosted woff2, субсеты latin+cyrillic):
  - заголовки — Playfair Display;
  - текст — Literata (запасной кандидат PT Serif — решаем на превью).
- Детали: малые прописные для рубрик и навигации, буквица в начале статьи, старостильные цифры в номерах механизмов, колонка ~65 знаков, интерлиньяж ~1.6, книжные поля, флероны-разделители (❦) между крупными частями статьи.
- Адаптивность: одна колонка, на телефоне поля уже, кегль не мельче 18px-эквивалента.
- Никаких тем/тумблеров: одна «бумажная» светлая тема.

## Хостинг, домен, публикация

- **Хостинг:** GitHub Pages, репозиторий `xenitch/practice-site`, ветка `main`, папка `/docs`. Файл `docs/CNAME` с `xenitch.ru`.
- **DNS:** домен куплен в Руцентре, делегирован на dns1/dns2.yandex.net (там настроена почта Яндекс 360 — MX `mx.yandex.net`, SPF; ящиком Ксения не пользуется, но сохраняем его живым). Настраиваем в панели, к которой есть доступ, — Руцентр: переключаем домен на DNS-серверы Руцентра и создаём записи:
  - `@ A` → 185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153
  - `www CNAME` → `xenitch.github.io.`
  - почтовые (перенос с Яндекса, чтобы ящик не умер): `@ MX 10` → `mx.yandex.net.`, `@ TXT` → `v=spf1 redirect=_spf.yandex.net`, плюс DKIM-запись `mail._domainkey TXT` (существует, публичный ключ снят с текущей зоны 2026-07-14: `v=DKIM1; k=rsa; t=s; p=MIGfMA0GCSqGSIb3DQEBAQUAA4GNADCBiQKBgQDatKNVtgqtrXo6nC1kKF7IbrxNoPTFBS0dtvbydfdVrWUvraANTPuUpJf6oIbLJfcO0clJsJ3U52Dff453qmbdDCPOOdFwOt5k5Q8+9u3THQW2YHbpBRG6pAN5/l8IUjwXVBepVplwtdj0sTGoyy1U6NonGpzqz3c7um3LpGBa1QIDAQAB`). DMARC-записи нет.
  - Запасной вариант (если в Руцентре что-то не так): не трогать делегирование и добавить записи сайта в панели Яндекс 360 (admin.yandex.ru → Домены → Управление DNS).

## Порядок работ

1. Скаффолд репозитория, `build.py`, шаблоны, стили; конвертация трёх статей и текстов главной/рамки.
2. Локальное превью для Ксении, итерации по дизайну.
3. Создание репозитория на GitHub (аккаунт `xenitch`), включение Pages — проверка на `xenitch.github.io/practice-site` (без домена).
4. DNS-записи в Яндексе (инструкция для Ксении по шагам), привязка домена, HTTPS.
5. Инструкция «как добавить статью» в README.
